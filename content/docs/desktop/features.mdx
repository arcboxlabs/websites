title: Features
description: An overview of what ArcBox Desktop can do.
---

<Cards>
  <Card title="Container Runtime" href="./containers">
    Run Docker containers with full API compatibility. Compose project grouping, real-time log streaming, integrated terminal, and live stats.
  </Card>
  <Card title="Linux Machines" href="./machines">
    Lightweight Linux VMs in seconds. Real kernel, SSH access, adjustable CPU/memory/disk. Ideal for systemd, custom kernel modules, or persistent Linux environments.
  </Card>
  <Card title="Sandboxes" href="./sandbox">
    Disposable development environments from templates. Node.js, Python, Rust, Go — start a preconfigured workspace and tear it down when done.
  </Card>
  <Card title="Kubernetes" href="./kubernetes">
    View and manage pods and services in your local cluster. Logs and terminal access work the same as with Docker containers.
  </Card>
  <Card title=".arcbox Domains" href="./domains">
    Every container and machine gets a DNS name automatically. Access services at `name.arcbox` from your Mac — zero configuration.
  </Card>
  <Card title="File Access" href="./file-access">
    Browse and manage files inside containers, images, and volumes directly from the Desktop GUI. Download or upload with a click.
  </Card>
  <Card title="Docker-Compatible CLI" href="./cli">
    Drop-in `docker` CLI replacement. Existing scripts, CI pipelines, and Compose files work without changes.
  </Card>
  <Card title="Benchmarks" href="./benchmarks">
    Built for Apple Silicon. ~300ms cold start. >90% native file I/O. Under 150 MB idle memory.
  </Card>
</Cards>

## Privacy

<Callout type="info">
  ArcBox runs entirely on your Mac. No account required. No telemetry by default. No feature gates on the free tier. Crash reporting via Sentry is opt-in.
</Callout>
